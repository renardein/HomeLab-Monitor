# HomeLab Monitor

[English](./README.md) | [Русский](./RU_README.md)

Локальная веб-панель для Proxmox и TrueNAS с отдельными макетами обычного и monitor-режима.

## Возможности

- **Proxmox VE**: узлы, хранилище, VM/CT, бэкапы и кворум на экране кластера
- **TrueNAS**: обзор системы CORE/SCALE
- **Проверки сервисов**: TCP, UDP, HTTP(S), SNMP и NUT
- **Монитор VM / CT**: иконки Iconify и цвета; показ или скрытие каждой карточки в режиме монитора
- **UPS и SNMP**: ИБП через NUT и сетевые SNMP-устройства
- **Плитки кластера**: до 12 KPI (сервис, VM/CT, SNMP, UPS или Speedtest) на экране «Кластер»
- **Время и погода**: часовой пояс и город; отдельная видимость на дашборде и в режиме монитора
- **Speedtest**: запуск по расписанию через CLI Ookla (см. [переменные окружения](#переменные-окружения))
- **Задачи бэкапов**: статус выполнения в режиме монитора
- **Метрики хостов**: температура CPU и скорость линка через [Node.js агент](./docs/proxmox_agent_manual_ru.md); обнаружение в кластере; установка и снятие агента по SSH из интерфейса
- **Режим монитора**: полноэкранные экраны, порядок, свайпы и стрелки; по умолчанию светлая тема
- **Telegram**: токен бота; при необходимости прокси к `api.telegram.org`; правила по типам событий; свои шаблоны сообщений с подстановками
- **Первый запуск**: мастер настройки или полный импорт JSON (тот же формат, что «Настройки» → экспорт)
- **Настройки**: пароль, время сессии, импорт/экспорт, хранение в SQLite
- **Отображение**: интервал обновления, свой CSS, пресеты стилей, пороги CPU/RAM на шкалах
- **Диагностика**: панель отладки, «О программе» (markdown), проверка релизов на GitHub
- **Локализация**: несколько языков
- **HTTPS**: опционально TLS (`certs/` или `SSL_*`); `TRUST_PROXY` за обратным прокси ([переменные окружения](#переменные-окружения))

## Быстрый запуск

Требования:

- Node.js 16+

Установка зависимостей:

```bash
npm install
```

Запуск приложения:

```bash
npm start
```

Режим разработки:

```bash
npm run dev
```

Агент host metrics:

```bash
npm run host-metrics-agent
```

Адрес по умолчанию:

```text
http://localhost:3000
```

(Порт задаётся в `.env` через `PORT`; значение `81` в старых инструкциях было лишь примером.)

## Переменные окружения

По желанию скопируй [`.env.example`](./.env.example) в `.env` в корне проекта. Переменные подхватываются при старте сервера в [`modules/config.js`](./modules/config.js) через `dotenv`.

| Переменная | Описание |
|------------|----------|
| **Сервер** | |
| `PORT` | Порт HTTP/HTTPS (по умолчанию `3000`). |
| `NODE_ENV` | `development` или `production` (влияет на флаг `Secure` у cookie вместе с TLS). |
| `BIND_HOST` | Адрес привязки (по умолчанию `0.0.0.0`). |
| `PUBLIC_URL` / `APP_URL` | Публичный URL без завершающего `/`, например `https://monitor.example.com` (логи / диагностика). |
| **TLS (HTTPS)** | |
| `SSL_KEY_PATH` | Путь к приватному ключу (PEM или DER PKCS#8/PKCS#1). |
| `SSL_CERT_PATH` | Путь к сертификату (PEM или DER `.cer`). |
| `SSL_CA_PATH` | Опционально: промежуточная цепочка. |
| — | Если не заданы, HTTPS включается автоматически при наличии **`certs/privkey.pem` + `certs/fullchain.pem`** или **`certs/private.key` + `certs/certificate.cer`**. См. [`certs/README.md`](./certs/README.md). |
| `TRUST_PROXY` | `1` или число хопов: за nginx/Caddy для корректных `X-Forwarded-*` и HTTPS. |
| `COOKIE_SECURE` | `0` / `1` — принудительно отключить или включить Secure у cookie; иначе авто (HTTPS или `production`). |
| **Хосты API по умолчанию** (если в UI ещё нет сохранённого подключения) | |
| `PROXMOX_HOST` / `PROXMOX_PORT` | Хост и порт API Proxmox (по умолчанию `10.200.0.1`, `8006`). |
| `TRUENAS_HOST` / `TRUENAS_PORT` | Хост и порт API TrueNAS (по умолчанию `10.200.0.2`, `443`). |
| **Прочее** | |
| `CORS_ORIGIN` | Origin для CORS у Express API (по умолчанию `*`). |
| `CACHE_TTL` | TTL кэша ответов API в секундах (по умолчанию `30`). |
| `DEFAULT_LANGUAGE` | Язык по умолчанию на стороне сервера (по умолчанию `ru`). |
| `LOG_DIR` | Каталог для `app.log` (по умолчанию `./data/logs`). |
| `LOG_MAX_BYTES` | Размер ротации логов (по умолчанию 5 МиБ). |
| `LOG_BACKUPS` | Сколько файлов ротации хранить (по умолчанию `5`). |
| `SPEEDTEST_CLI` / `SPEEDTEST_PATH` | Путь к бинарнику Speedtest/Ookla CLI. |
| **Агент host metrics** (только для `npm run host-metrics-agent`, не основное приложение) | |
| `HOST_METRICS_AGENT_HOST` | Адрес прослушивания (по умолчанию `0.0.0.0`). |
| `HOST_METRICS_AGENT_PORT` | Порт (по умолчанию `9105`). |
| `HOST_METRICS_AGENT_BASE_PATH` | Базовый HTTP-путь (по умолчанию `/host-metrics`). |
| `HOST_METRICS_SENSORS_BIN` / `HOST_METRICS_ETHTOOL_BIN` | Пути к `sensors` и `ethtool`. |
| `HOST_METRICS_COMMAND_TIMEOUT_MS` | Таймаут команд (по умолчанию `3000`). |

## Настройка

1. Открой приложение в браузере. При первом запуске (или после полного сброса настроек) пройди **мастер начальной настройки** или импортируй JSON-бэкап.
2. Если не сделал это в мастере, в `Settings` добавь подключение к Proxmox и/или TrueNAS.
3. Настрой сервисы, VM / CT, UPS, SNMP-устройства, Speedtest и host metrics.
4. При необходимости измени порядок экранов monitor mode, темы, параметры отображения и видимость экранов.
5. При необходимости настрой **Telegram** в разделе настроек.
6. При необходимости включи пароль на настройки и используй импорт / экспорт.
